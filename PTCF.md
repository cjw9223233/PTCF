我幫你用 **Persona / Task / Context / Format（PTCF）四大核心要素** 重新分類、列點，並整理成清晰表格，讓它變成一套可重複使用的提示詞架構系統。

# 一、四大核心要素總覽

高效提示詞 =

> **Persona（角色） + Task（任務） + Context（情境） + Format（格式）**

---

# 二、依四要素分類整理

---

## 一）Persona（角色）

👉 定義 AI 應該用什麼「專業身份」思考

### 你常用的角色類型：

| 類型   | 範例                   |
| ---- | -------------------- |
| 技術類  | 程式設計師 / PM / 架構師     |
| 投資金融 | 證券投資分析師 / CFA / CIIA |
| 顧問型  | 資深顧問                 |
| 教學型  | 初學者友善講師              |
| 醫療健康 | 減重醫師 + 營養師           |
| 分析型  | 第一性原理拆解者             |
| 批判型  | 反方辯證者                |

### 你常見的 Persona 指令語句：

* 你是一位【角色】
* <角設> 你是一位…
* 請用初學者友善方式教我
* 請挑戰我的觀點
* 請用第一性原理拆解

👉 這些全部屬於 **Persona 控制器**

---

## 二）Task（任務）

👉 AI「要做什麼」

### 你常用任務類型：

### 1️⃣ 摘要整理型

* 幫我總結摘要
* 幫我整理重點
* 抽取核心觀點
* 產出簡報摘要
* 列 5-10 個關鍵字
* 列點整理

### 2️⃣ 優化改寫型

* 修改語意更順
* 邏輯更清晰
* 標準英文修正

### 3️⃣ 分析拆解型

* 找底層邏輯
* 第一性原理拆解
* 這句話有沒有道理？
* 依據理論為何？
* 提供實例

### 4️⃣ 批判挑戰型

* 挑戰我的觀點
* 提出不同角度

### 5️⃣ 教學型

* 用初學者方式教我
* 用比喻 + 步驟 + 常見誤區

### 6️⃣ 專案解析型

* 這專案做什麼？
* 使用哪些技術？
* 進入點在哪？
* 解釋夾結構
* 最佳實踐
* 流程圖 / 架構圖

### 7️⃣ 程式技術型

* 說明並註解程式碼
* 列出步驟 + 理由
* 強化哪些環節

👉 這全部屬於 **Task 層**

---

## 三）Context（情境）

👉 提供必要背景資訊

### 你常用 Context 模板：

```
背景：
【必要上下文】
```

```
資料：
<<<
貼上內容
>>>
```

```
限制條件：
【字數 / 禁用內容 / 語氣】
```

### 典型 Context 類型

| 類型    | 範例              |
| ----- | --------------- |
| 專案背景  | 專案開發            |
| 投資標的  | 3481 群創光電       |
| 技術環境  | Docker / Ubuntu |
| 輸出限制  | 禁用簡體字           |
| 使用者層級 | 初學者             |

👉 Context 是讓 AI 不亂猜的關鍵。

---

## 四）Format（輸出格式）

👉 規定「怎麼呈現」

你其實非常常用 Format 控制。

### 常見格式要求：

* 條列式
* 表格
* JSON
* 段落
* 表格 + 摘要
* 流程圖
* 架構圖
* README.md
* Markdown 文件

### 你的格式控制語句：

* 幫我列點
* 整理成表格
* 附上摘要
* 產出流程圖
* 寫成 README.md

👉 這全部屬於 Format 層。

# 三、完整分類總表

| 類別      | 核心問題       | 你常用語句類型                  | 目的     |
| ------- | ---------- | ------------------------ | ------ |
| Persona | 用什麼專業身份思考？ | 你是一位… / 初學者教學 / CFA      | 控制思維框架 |
| Task    | 要完成什麼？     | 摘要 / 分析 / 挑戰 / 教學 / 專案解析 | 控制行為   |
| Context | 在什麼情境下？    | 背景 / 資料 / 限制條件           | 降低模糊   |
| Format  | 怎麼輸出？      | 表格 / 條列 / JSON / README  | 提高可用性  |

# 四、你這套 Prompt 的底層邏輯

你其實已經在做：

> 把「思考流程」結構化

你所有提示詞都圍繞這四件事：

1. 定義思考角色
2. 明確任務
3. 補充上下文
4. 控制輸出格式

這正是高效提示詞的核心結構。

---

# 五、進階洞察（給你一個更高階視角）

你的提示詞其實可以升級成三層系統：

### 第一層：思考層（Persona）

決定模型的「大腦」

### 第二層：操作層（Task）

決定模型「做什麼」

### 第三層：表達層（Format）

決定模型「怎麼說」

Context 是讓這三層運作精準的「燃料」。

> **角色 × 任務 × 情境 × 格式 = 穩定高品質輸出**
